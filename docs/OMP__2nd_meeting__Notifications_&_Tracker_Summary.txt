Notifications & tagging
Tagging was added so users receive in-app notifications via the platform bell when @-mentioned.
Current behavior: tagging produces platform-only alerts (no automatic email) to avoid inbox flooding;
team discussed options for optional email fallback.
Suggested conventions: use @name for individuals, @team / @all for group-wide messages (but use
sparingly).
Mobile PWA notifications discussed: ideal is a gentle transient notification that pops and vanishes (like
map directions), avoiding accumulating hundreds in the bar.
Forward view: define notification types and rules (personalized vs group) and train the team on
@-mention discipline to keep noise low.

Progress tracking, KPIs & metrics
There is currently no semantic rule that automatically rolls progress numbers up from activity; users
saw zero progress values in some objectives.
Ideas discussed: track time spent/activities and translate to incremental progress (e.g., Jake working
20 minutes increments progress bar).
Need clarity on whether progress updates are manual or computed automatically and what
baseline/target metrics should display.
Next: decide progress-tracking rules (manual update vs auto-rollup) and expose baseline/target fields
in the objective view.
Files, attachments & previews
Team validated preview behavior: attachments should preview without forcing downloads to avoid
cluttering user machines.
Current issue: clicking an objective can surface historical attachments for the whole group, potentially
confusing who attached which file.
Access control: sensitive files should only be visible to users who are in the relevant group/chat;
otherwise avoid including them in group attachments.
Next: tighten attachment UX so a single-click opens the intended file preview and clarify group vs
private attachments.
Mobile UX & bugs

back-navigation behavior on mobile.
Reported issues: rotating Android phones to landscape produced blank/unclickable views in some
flows; Safari share/add-to-home instructions vary by OS (iOS vs Android).
PWA behavior: team discussed installing the tracker as a PWA (add to home screen) for mobile
push-like notifications without App Store distribution and avoiding Apple fees (33% referenced).
Next: reproduce rotation bug on Android, test PWA install flow for both iOS and Android, and
document user steps to add to home screen.
Product strategy & UX tradeoffs
Notifications are viewed as the low-hanging fruit — immediate leverage to reduce friction and
improve adoption.
Tension: notifications must be relevant and actionable (keep users in flow) but not so frequent they
cause alert fatigue and abandonment.
Team also discussed experimental AI features (auto-prompts, suggested actions); Andrew temporarily
toggled AI features off to avoid overwhelming the demo.
Forward view: define a lightweight notification taxonomy, train users on mention conventions, and
iterate on a gentle mobile notification UX that balances immediacy and noise.
