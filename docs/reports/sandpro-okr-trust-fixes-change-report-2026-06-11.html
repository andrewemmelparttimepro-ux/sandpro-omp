Executive Summary

This release protects user trust in the new OKR framework. The app now avoids confidently labeling ordinary operational work as Projects or KRs, shows uncertain classifications as muted review chips, keeps OKR-only filters out of the default list view, and requires senior-management signoff only when the project budget reaches the default $25,000 threshold.

What Changed

  • Narrowed project keywords to stage-gate language such as project, prototype, pilot, R&D, research, feasibility, new product, and stage gate.
  • Keyword matches now become review suggestions instead of authoritative project labels.
  • Renamed Needs Assessment to Unclassified and rendered uncertain labels as muted dashed chips in tree and list views.
  • Changed senior-management project signoff to trigger only at/above $25,000 or when explicitly flagged.
  • Scoped OKR-specific filter selects to tree view, while keeping active dashboard lens filters visible when needed.

Why It Matters

  • Jake and Tim see the app admitting uncertainty instead of pretending the classifier knows more than it does.
  • Rollup trust improves because questionable work is flagged for human review before it affects OKR interpretation.
  • The Objectives list regains a focused daily-working toolbar, while the OKR tree keeps the deeper planning controls.
  • The approval gate no longer overreaches by requiring senior management on every small project.
  • The framework remains a working prototype for Jake's approval-matrix decisions, not a hard-coded final answer.
