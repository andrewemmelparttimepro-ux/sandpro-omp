NCR / CAPA Production Build Plan

Move the NCR side from a report tracker into a quality-manager-grade corrective action workflow.

Next target: Tim-ready NCR baseline

Target NCR Lifecycle

1. Draft / IntakeReport number, event, location, severity, people involved, attachments.
2. SubmitFreeze core facts, notify owner, start audit trail.
3. ContainmentHold, rework, repair, scrap, return, or use-as-is decision.
4. Root CauseCause codes, analysis notes, recurrence risk.
5. Corrective ActionAssigned tasks, due dates, objective handoff, proof evidence.
6. Effectiveness CheckVerifier confirms action worked and recurrence risk is reduced.
7. ApprovalQuality/manager signoff before true closure.
8. Closed / ArchivedLocked record, searchable history, export-ready packet.

Holes To Seal

Closure Discipline

Replace one-click close with owner, verifier, and approval gates.

Containment / Disposition

Add hold, rework, repair, scrap, return, concession, use-as-is.

Native Action Items

Corrective tasks inside NCR plus objective handoff when needed.

Evidence

Photos, PDFs, inspection records, customer docs, signed approvals.

Audit Trail

Who changed what, when, from old value to new value.

Trend Detection

Repeat issue, root cause, department, customer, equipment, aging.

Recommendation
Build NCR as a controlled corrective-action system, not just a table of reports.
Benchmark logic aligned to common CAPA expectations: document nonconformity, investigate cause, implement action, verify effectiveness, preserve records.
