# SandPro OMP NCR Audit - 2026-06-02

## Executive Summary

The NCR Tracker is now production-validated for the shipped baseline workflow: view/search/filter NCRs, create a new NCR, review event/root-cause/corrective-action/follow-up details, close an NCR, reopen an NCR, create a linked objective from an NCR, and view the NCR page on mobile without crop/overflow.

One real bug was found and fixed during the audit: after marking an NCR closed while filtered to Open, the selected report disappeared and the Reopen action was no longer visible. The detail panel now keeps the selected NCR visible even when its status moves outside the active filter, so users can immediately reopen a mistaken closure.

## Baseline Opinion

The shipped NCR tracker is stable enough as a baseline tracker. It is not yet a complete quality-management/CAPA module. For Tim's review, that distinction matters: the current system reliably tracks and turns NCRs into objectives, but it does not yet cover every workflow a quality specialist may expect.

Recommended Tim feedback areas:

- Whether NCRs need post-create edit controls for quality managers.
- Whether NCRs need attachments/evidence files.
- Whether there should be assigned owner, disposition, containment, verification, and approval fields.
- Whether closeout should require corrective-action verification before closure.
- Whether imports/exports should match SandPro's official NCR spreadsheet/report format.
- Whether critical NCRs should trigger automatic push/email escalation rules.
